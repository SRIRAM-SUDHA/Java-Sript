# PHASE 9 — Testing & Production Readiness

## Goal
Be confident shipping Angular: unit testing, high-signal best practices, and avoiding common production pitfalls.

## Topics
1. **Unit Testing**
   - `TestBed`
   - Component tests
   - Service tests
   - Mocking dependencies
2. **E2E & Best Practices**
   - High-level E2E approach (tooling varies by project)
   - Angular best practices
   - Common anti-patterns
   - Migration tips from React

## Hands-on (Do This)
- Write unit tests for:
  - one service
  - one “smart” component
  - one reusable UI component
- Add a small “testing utilities” area for common mocks/builders.
- Create a short checklist you’ll follow before deploying.

## Exit Criteria
- You can write and maintain tests without fighting the framework.
- You can review Angular code and spot common issues early.

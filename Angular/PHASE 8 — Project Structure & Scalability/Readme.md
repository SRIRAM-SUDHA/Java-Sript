# PHASE 8 — Project Structure & Scalability

## Goal
Structure Angular projects like teams do: clear boundaries, scalable organization, and pragmatic state management.

## Topics
1. **Angular Folder Architecture**
   - Core / Shared / Feature patterns
   - Smart vs dumb components
   - Feature isolation
   - Monorepo mindset (high-level)
2. **State Management (Optional but Useful)**
   - Service-based state patterns
   - ComponentStore (conceptual)
   - NgRx overview (conceptual)

## Hands-on (Do This)
- Restructure your app into feature areas (each feature owns its UI + services + routes).
- Create shared UI components and utilities with clear rules (no feature imports inside shared).
- Optionally prototype a small store pattern for a complex feature.

## Exit Criteria
- You can explain the project layout and where new code should live.

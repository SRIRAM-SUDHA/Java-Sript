# PHASE 4 — Forms (Angular’s Power Area)

## Goal
Become strong with Reactive Forms (the enterprise default) and understand when template-driven forms are appropriate.

## Topics
1. **Template-Driven Forms (Quick)**
   - Basic understanding only
   - When NOT to use them
2. **Reactive Forms (Critical)**
   - `FormGroup`, `FormControl`, `FormBuilder`
   - Built-in validators + custom validators
   - Dynamic forms
   - Form state management patterns

## Hands-on (Do This)
- Build a multi-step “Profile Editor” form:
  - Cross-field validation
  - Dynamic add/remove controls (e.g., skills list)
  - Show errors only when appropriate (touched/dirty)

## Exit Criteria
- You can build non-trivial forms without brittle template logic.

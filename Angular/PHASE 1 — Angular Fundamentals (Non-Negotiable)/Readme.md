# PHASE 1 — Angular Fundamentals (Non-Negotiable)

## Goal
Become productive with Angular’s core building blocks (components + templates) and understand how modern Angular apps are structured.

## Topics
1. **Angular Architecture Overview**
   - What Angular is/is not (framework mindset)
   - How Angular apps boot
   - Angular CLI philosophy
2. **Modules vs Standalone Components (Modern Angular)**
   - `NgModule` basics (legacy but still seen)
   - Standalone components (Angular 14+)
   - How Angular resolves dependencies
3. **Components (Core Building Block)**
   - `@Component` decorator + metadata
   - Component class vs template
   - Change detection basics
   - Component communication: `@Input()` / `@Output()`
4. **Templates & Data Binding**
   - Interpolation, property binding, event binding
   - Two-way binding
   - Template reference variables
5. **Directives (Structural & Attribute)**
   - Built-ins: `*ngIf`, `*ngFor`
   - Why `*` exists (desugaring)
   - Writing custom directives
   - Directive vs component decision
6. **Pipes**
   - Built-in vs custom pipes
   - Pure vs impure pipes
   - Performance implications

## Hands-on (Do This)
- Create a small “Dashboard” app with 3–5 components and reusable UI pieces.
- Use at least: `*ngIf`, `*ngFor`, `@Input()`, `@Output()`, a custom pipe, a custom directive.

## Exit Criteria
- You can explain and implement: component ↔ template, bindings, directives, pipes, and basic app structure.
- You can read typical Angular code without getting lost in decorators/metadata.
